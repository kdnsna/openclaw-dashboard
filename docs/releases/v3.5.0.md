# 小锤子监控台 v3.5.0 发布说明

发布日期：2026-03-30

## 这次升级是什么

`v3.5.0` 是小锤子监控台一次偏正式、偏产品化的大版本整理。

它不是一次“修复若干问题、优化若干体验”的例行更新，而是一轮把监控台从“工程上可用”继续往“产品上更成熟”推进的升级。这个版本集中处理的，是那些平时最容易影响真实使用感、却经常不会被一句“功能已经有了”覆盖掉的问题：

- 首屏启动时像不像坏了
- 数据还没到时界面会不会误报离线
- 会话、ACP、Cron 这些内容是不是只有写代码的人才看得顺
- 某个局部接口挂了以后，页面是不是还像个完整产品
- 窄屏和手机下，到底是“能打开”，还是“真的能用”

从这个角度看，`v3.5.0` 的价值不在于某一个单点功能，而在于它把很多“细碎但决定成熟度”的地方补齐到了一个新的层级。

## 核心升级一：首屏启动编排重做

这一版重做了前端首轮数据进入方式：

- 首次进入页面时，并行拉取 `metrics / acp-workflow / task-flow / task-events`
- 用 `allSettled` 协调首轮资源
- 首轮完成后再继续接入 WebSocket 实时流
- 在前端暴露 `bootState`、`resourceStates`、`refreshAll`

过去页面刚打开时，用户经常会遇到一种很尴尬的状态：其实只是数据还没到，但界面视觉上已经有点像断线或离线。对监控台来说，这种误导很伤体验，因为用户会被迫先分辨“到底是系统坏了，还是只是页面在启动”。

现在页面有了更清晰的节奏：

1. 正在连接监控源
2. 正在同步首轮快照
3. 首轮数据开始进入
4. 进入实时态

这让首屏体验从“容易误会”变成了“能被理解”。

## 核心升级二：服务端首次快照提前

服务端现在会在启动后立即开始第一次快照采集，不再保留此前多秒级的等待窗口。

这个变化的产品意义非常直接：

- 第一次有效数据出现得更快
- 空档期更短
- 假离线感更弱
- 整个监控台更像一个真正“启动后马上开始工作”的系统

对监控产品来说，用户对“第一眼”的判断非常重要。`v3.5.0` 这次就是把这种第一眼体验尽量往前推。

## 核心升级三：业务命名与展示层统一

这次专门补了一层展示表达层，统一整理了：

- `dashboard-task-summarizer`
- Cron 任务名回映
- 微信直连会话识别
- ACP 会话展示层
- ACP 指标口径文案

升级前，有些区域仍然保留较强的工程字段感。它们虽然“技术上正确”，但对真实阅读并不友好。升级后，会话和 ACP 区域会更强调：

- 名称是否好理解
- 业务含义是否足够明确
- 文案是否像产品，而不是像日志片段

这会直接提升看板的理解速度，也让检修页在打开时不再那么“只适合作者本人”。

## 核心升级四：卡片状态体系统一

这一版统一补齐了卡片状态：

- `loading`
- `error`
- `stale`
- `retry`

很多产品在正常时做得不错，但一旦进入加载、波动、失败、陈旧状态，界面就会立刻露出“其实没有认真设计过这些情况”的破绽。监控台尤其如此，因为它天然就会面对不稳定和局部失败。

`v3.5.0` 之后，这些状态不再是零散处理，而是作为一套统一语言落到多张核心卡片中。结果就是：

- 页面更有边界感
- 局部失败不再像整页崩溃
- 用户更容易理解当前状态
- 监控台更像成体系产品，而不是理想路径演示页

## 核心升级五：异常反馈和重试能力增强

这次版本不仅做了正常路径验证，也专门做了故障注入回归。

在模拟 `task-events` 返回 `503`、同时禁用实时链路的情况下，页面能够正确显示：

- “任务事件暂不可用”
- 对应的局部错误提示
- `重试` 按钮

这一点很关键。因为真正成熟的监控台，不是只在数据都正常时显得顺眼，而是在出现异常时仍然能保持：

- 信息准确
- 反馈克制
- 局部边界明确
- 恢复入口清晰

这也是为什么这次升级更值得被当成一次正式版本发布，而不是一个零散修补集合。

## 核心升级六：多视口和真实浏览器表现继续打磨

本轮对真实浏览器表现做了完整回归：

- `1600x1200` 桌面宽屏通过
- `390x844` 手机宽度通过
- `Overview / Inspect` 两页复查通过
- 手机宽度下无横向溢出
- `console` 正常路径无报错
- `network` 4 个核心接口正常返回 `200`

这意味着小锤子监控台在“可以访问手机”之外，又往“手机上也不容易崩、读起来也更顺”迈进了一步。

## 你会感知到什么变化

如果你是第一次打开这个项目，你会感知到：

- 首屏不慌张
- 信息更有层级
- 文案更像产品
- 异常时不至于一头雾水
- 手机上也能用来快速看一眼

如果你已经用过之前的版本，你会更明显感受到：

- 页面启动不再容易被误判成离线
- 会话 / ACP / Cron 的理解门槛降低了
- 局部接口挂掉时页面终于有了体面的反馈
- GitHub 首页截图和文档终于和当前真实界面同步了

## 验证结果

本次发布前已完成：

- 构建通过
- TypeScript 校验通过
- 浏览器真实访问验证通过
- 正常路径下 `console` 无报错
- 正常路径下 4 个核心接口返回 `200`
- 故障注入回归通过，错误提示和重试能力符合预期

## 已知后续方向

`v3.5.0` 并不是终点，后续仍然值得继续推进：

- ACP 与普通 sessions 的跨资源关联
- ACP 更细的项目级命名
- 按资源粒度的重试能力
- 更完整的长文本展开与深入诊断交互

## 结语

如果要用一句话描述这个版本，我会说：

`v3.5.0` 让小锤子监控台从“已经能用”更进一步，走向“更稳、更顺眼、更像一个认真打磨过的产品”。

这也是为什么这次发布值得被写得更完整一些。它不是一次简单的版本号跳动，而是一轮明显的成熟度提升。
