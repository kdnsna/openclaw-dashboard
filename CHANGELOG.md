# Changelog

本项目遵循语义化版本（SemVer）。

## v3.0.0 - 2026-03-25

### 里程碑
- 完成一次面向产品结构的重大升级：监控台从“平铺式实时看板”升级为“运营驾驶舱 + 检修入口”。
- 明确与 OpenClaw 官方 Dashboard 的职责分工：3210 负责上层运营视图与行动摘要，18789 负责底层系统真相与原厂诊断。

### 新增
- 新增总览 / 检修双视图切换，支持按场景切换日常查看和深入排查。
- 新增 AttentionCard，用规则化方式汇总“当前最值得关注”的事项。
- 新增 SystemSummaryCard，集中展示系统健康、连接状态和摘要级运行信息。
- 新增 AutomationStatusCard，接入自动化任务状态聚合结果。
- 新增 OfficialDashboardLinksCard，提供官方 Dashboard 深链入口。
- 新增 AcpWorkflowSummaryCard，在总览页展示 ACP 工作流摘要。
- 后端新增 `automation-status.ts` 与 automation / officialDashboard 数据输出结构。

### 增强
- `SessionsCard` 升级为“会话工作台”，增加优先级排序、风险标记和摘要条。
- `TaskLogCard` 升级为“任务时间线”，增加持续时长、最近活动时间和需关注状态识别。
- `AcpWorkflowCard` 升级为“ACP 工作流检修”视图，增加阶段、时长、慢工具、开始/更新时间等更多诊断细节。
- `App.tsx` 重构页面布局，按总览 / 检修组织现有卡片与新卡片。
- `format.ts`、`types.ts` 扩展格式化函数与类型定义，支持自动化状态、官方链接与持续时长展示。

### 样式与体验
- 前端样式进行大规模重构，新增双视图 tabs、总览/检修布局、摘要卡、风险 pill、自动化区块和更多响应式细节。
- 首页信息密度从“平铺卡片”调整为“先判断，再深入”的结构。
- 检修视图增强了会话、任务和 ACP 的可读性，更适合排查与回看。

### 说明
- 本次版本属于结构级大升级，变更范围覆盖前后端数据结构、页面布局与信息分层。
- v2.x 时代的“宽屏浏览器下偶发卡片大小与风格错位”问题已通过新布局体系明显缓解，但后续仍可继续做视觉细节打磨。

## v2.0.0 - 2026-03-23

## v1.2.2 - 2026-03-22

## v1.1.0 - 2026-03-21

### 修复
- 调整 Token 卡片文案，明确区分“近 30 天 / 账单口径”和“累计账本（全历史）”
- 修正累计账本摘要中的总 Tokens 展示，避免把“输入+输出”误显示成“累计总量”
- 增加累计输入输出与累计缓存读取拆分，减少口径误解

### 说明
- 本版本是对监控台 Token 统计展示逻辑的修正版本
- 后续将继续排查 3 月 19 日与 3 月 20 日历史数据补算问题

## v1.0.0 - 2026-03-20

### 里程碑
- 监控台进入首个正式可用版本
- 支持桌面端使用，并开始支持移动端 / PWA 形态

### 新增
- 独立 Git 仓库与 GitHub 私有仓库
- 项目目录整理与 README 完善
- 中文化监控台界面
- 长期账本能力接入（后端累计统计）
- 长期账本摘要整合进 Token 卡片
- 成本统一改为人民币显示
- 基础 PWA 能力：manifest、service worker、安装图标位
- 移动端基础适配：viewport / safe-area / 手机访问基础支持
- 支持局域网访问监控台，可通过同一网络下的手机直接打开

### 修复
- 恢复主版面 6 卡布局
- 修复多次迭代中的前端编译问题
- 统一成本显示逻辑，避免美元与人民币混用

### 说明
- 当前版本的移动端支持为第一阶段：
  - 可作为 Web App / PWA 使用
  - 已具备添加到主屏幕的基础能力
  - 后续仍可继续优化手机端交互与版面
